\section{Introduction}
%=============================================================================

Large language models (LLMs) \citep{brown2020language, chung2024scaling} have enabled autonomous agents capable of reasoning, tool use, and multi-turn interaction with complex environments \citep{yao2023react, jimenez2023swe, su2025learnbyinteract}. Reinforcement learning (RL) with verifiable rewards (RLVR) has further advanced these capabilities, enabling agents to discover effective policies through direct environment interaction \citep{shao2024deepseekmath, guo2025deepseek, jin2025search, xue2025simpletir}.

While RL provides a powerful learning signal, each training step is expensive: every GRPO step generates $G$ multi-turn rollouts per question, each involving multiple rounds of model inference and environment interaction. This computational cost motivates maximizing the utility of each rollout, extracting not only policy gradients but also insights about what strategies succeed and fail.

A promising direction for improving agent efficiency is \textit{skills}: modular, reusable behavioral patterns that guide agent actions in specific situations. Anthropic's Skill Creator \citep{anthropic2025skillcreator} pioneered the automated creation of such modular skills from agent experience, demonstrating that structured skill libraries can dramatically improve agent capabilities. SkillRL \citep{xia2025skillrl} distills teacher-model knowledge into skill banks that augment RL training. Other approaches accumulate experiential knowledge through verbal reinforcement \citep{shinn2023reflexion} or episodic learning \citep{zhao2024expel}. However, these approaches treat skills as \textit{static} artifacts, created before or outside the RL training loop and then frozen during policy optimization.

The fundamental limitation of static skills mirrors that of static instructions in RLVR \citep{zhou2025inspo}: the optimal skill set is not known \textit{a priori} and changes as the agent's policy improves. A skill that scaffolds a weak policy (e.g., ``always search before answering'') may constrain a stronger policy that has internalized this behavior. Conversely, a skill addressing subtle failure modes only becomes relevant after the policy has mastered basic competencies. Static skill banks, whether distilled from teacher models \citep{xia2025skillrl} or accumulated from fixed experience \citep{zhao2024expel, shinn2023reflexion}, cannot adapt to these shifting needs and risk either under-guiding or over-constraining the evolving policy \citep{soylu-etal-2024-fine, liu2025understanding}. This creates a core tension: \emph{skills guide what can be instructed; policy learns what must be discovered}.

To bridge this gap, we propose \ours, an \textbf{RL}-guided \textbf{S}kill \textbf{C}reator for agentic skill-policy co-evolution. \ours~integrates skill creation and evolution as dynamic components of the RL loop, delivering two major innovations. First, we introduce an \textit{assertion-driven skill version creator} that evolves the skill library through structured versions. A rule-based assertion grader provides quantitative failure diagnosis, an LLM-based analyzer identifies representative failure patterns, and a two-step version creation process (analyst $\to$ author) proposes targeted changes to the skill library---adding new skills, modifying existing ones, or deleting obsolete ones---each paired with \textit{trigger conditions} that determine when skills activate during rollouts. This enables situational guidance without prompt bloat. Second, we propose an \textit{RL-guided skill evolution mechanism via Thompson Sampling} \citep{thompson1933likelihood}, where GRPO rollouts serve \textbf{triple duty}: policy optimization via standard gradients, failure collection for skill creation via a bounded reservoir, and bandit evaluation for skill evolution via Thompson Sampling. Each proposed version of the skill library is tested as a bandit decision by stochastically assigning each episode to either the new or current version, with Beta-Binomial posteriors providing principled, sample-efficient accept/reject decisions. Zero additional rollouts are required beyond standard GRPO training.

We validate \ours~through extensive experiments on three diverse agentic benchmarks, WebShop \citep{yao2022webshop}, multi-hop question answering with search tools, and ALFWorld \citep{shridharalfworld}, across three model scales (4B, 8B, 14B parameters). \ours~achieves substantial convergence speedup over GRPO and static skill baselines, with ablation studies confirming the contribution of each component. The entire skill creation and evolution process incurs only marginal computational overhead: a few LLM calls per evolution cycle, with no additional rollouts.

\section{Preliminaries}
%=============================================================================

\subsection{Multi-Turn Agentic RL}
We consider an LLM-based agent with policy $\pi_\theta$ that interacts with an environment over multiple turns via a tool engine $\mathcal{T}$. Given a task $q \sim \mathcal{D}$ and a set of active skills $\mathcal{S}_{\text{active}} \subseteq \mathcal{S}$ injected into the prompt, the agent generates a trajectory $\tau$ of interleaved actions and observations. The learning objective is to maximize the expected reward under a rule-based verifier $r(\tau) \in \{0, 1\}$:
\begin{equation}
    \max_{\pi_\theta} \mathbb{E}_{q \sim \mathcal{D},\, \tau \sim \pi_\theta(\cdot|q, \mathcal{S}_{\text{active}};\mathcal{T})} \left[ r(\tau)
    \right].
    \label{eq:rl_objective}
\end{equation}

\subsection{Group Relative Policy Optimization (GRPO)}
Group Relative Policy Optimization (GRPO) \citep{shao2024deepseekmath} is a policy optimization method that derives from Proximal Policy Optimization (PPO) \citep{schulman2017proximal} and obviates the need for a separate value function, instead using group-wise sampling to estimate the advantage function $\hat{A}$ based on relative performance within a group $G$. For each task $q\sim\mathcal{D}$, GRPO samples a group of trajectories $\{\tau_i\}_{i=1}^G$ from the old policy $\pi_{\theta_{\text{old}}}(\cdot \mid q, \mathcal{S}_{\text{active}}; \mathcal{T})$ and optimizes:
\begin{equation}
\begin{aligned}
&\mathcal{J}_{\text{GRPO}}(\theta) = \mathbb{E}_{\tau \sim \pi_{\text{old}}} \Bigg[ \frac{1}{G} \sum_{i=1}^G \frac{1}{\sum_t \mathcal{I}_{i,t}} \sum_{t=1}^{|\tau_i|}\mathcal{I}_{i,t} \cdot \min \left( \rho_{i,t}\hat{A}_{i}, \text{clip}(\rho_{i,t}, 1{-}\epsilon, 1{+}\epsilon) \hat{A}_{i} \right)  - \beta \mathbb{D}_{\text{KL}} \Bigg],
\end{aligned}
\label{eq:grpo}
\end{equation}
where $\rho_{i,t}(\theta) = {\pi_\theta(\tau_{i,t}|\tau_{i,<t})}/{\pi_{\theta_{\text{old}}}(\tau_{i,t}|\tau_{i,<t})}$ is the token-level importance ratio, $\hat{A}_i = ({r_i - \text{mean}(\{r_j\}_{j=1}^{G})})/{\text{std}(\{r_j\}_{j=1}^{G})}$ is the group-normalized advantage, $\epsilon$ is the clip ratio, and $\beta$ controls the KL divergence penalty $\mathbb{D}_{\text{KL}}(\pi_\theta \| \pi_{\text{ref}})$ with respect to the reference policy. The indicator $\mathcal{I}_{i,t}$ masks out environment observation tokens so that gradients are computed only over the agent's own tokens \citep{jin2025search, xue2025simpletir}.

\section{RL-Guided Skill Creation (\ours)}
\label{sec:method}
%=============================================================================

The conventional RLVR approach treats any auxiliary guidance, whether instructions \citep{zhou2025inspo} or skills \citep{xia2025skillrl}, as a static, pre-defined input. This constrains the agent's exploration and may result in suboptimal convergence, as the ideal guidance changes with the evolving policy. \ours~re-frames skill design as a \textit{dynamic} and \textit{integral} component of the RL loop, enabling skills and policy to co-evolve online without any additional rollouts. The key observation is that standard GRPO rollouts already provide all the signal needed: we describe this \textit{triple-duty} design in \S\ref{sec:design}, then detail the two components it connects---the skill creator pipeline (\S\ref{sec:skill_creator}) and the Thompson Sampling-based skill evolution mechanism (\S\ref{sec:skill_evolution}).

\subsection{Triple-Duty Rollouts: Connecting Skill Creation and Evolution}
\label{sec:design}

Recall that each GRPO step samples a group of $G$ rollouts from the same task $q$ under the current policy $\pi_{\theta}$. A key observation is that rollouts within the same group share the same task input and policy, differing only in their stochastic trajectories---this provides a natural, controlled setting for comparing two skill configurations. Rather than splitting across \textit{tasks} (which introduces high variance when the number of episodes is small, as is typical in multi-turn agentic RL, e.g., $G = 16$), we split across \textit{rollouts within each group}: some rollouts use the candidate skill version $\mathcal{V}_{\text{new}}$ while others use the current version $\mathcal{V}_{\text{old}}$, with the allocation determined by Thompson Sampling (\S\ref{sec:skill_evolution}). This within-group split provides a fairly controlled comparison where the only difference is the skill configuration.

Building on this, we design each GRPO rollout to simultaneously serve three purposes: (i)~\textbf{policy optimization}---the reward signals provide the GRPO advantage for updating $\theta$ via Eq.~(\ref{eq:grpo}), which remains the primary purpose of the rollout; (ii)~\textbf{failure collection}---trajectories with $r(\tau) < 1$ are added to a bounded reservoir $\mathcal{R}$, providing the raw material for the skill creator pipeline (\S\ref{sec:skill_creator}); and (iii)~\textbf{skill evaluation}---each rollout's outcome under its assigned skill version updates the Thompson Sampling posteriors for accepting or rejecting the candidate version (\S\ref{sec:skill_evolution}). The skill creation and evolution thus operate entirely within the existing GRPO training loop, with the only additional cost being periodic LLM calls for the skill creator (fewer than 30 API calls across a full training run).

\subsection{Skill Creator Pipeline}
\label{sec:skill_creator}

We first define the \textit{skill module} as the atomic unit of guidance in \ours. A skill $S_k = (c_k, \text{trig}_k)$ consists of content $c_k$ (a structured instruction snippet) and a trigger condition $\text{trig}_k$ that determines when the skill activates during a rollout. Trigger conditions are parameterized by type and pattern, allowing the skill optimizer to choose from several activation modes (e.g., every step, only at the beginning, or only when a specific action pattern is observed). The skill bank $\mathcal{S} = \{S_1, \ldots, S_K\}$ collects all active skills. Skills are \textit{modular}: each targets a specific failure mode and activates only when its trigger condition is met, enabling compositional guidance without prompt bloat.

The skill creator pipeline adapts the Skill Creator paradigm, originally designed for human-in-the-loop development, for automated use within RL training. The input is failed GRPO rollouts collected over time into a bounded failure reservoir $\mathcal{R}$. The pipeline operates in three stages, triggered periodically (every $K_e$ GRPO steps) when the system is not already testing a candidate skill.

\rparagraph{Assertion Grading and Analysis}
The assertion grader $\mathcal{G}$ maintains a dynamic set of rule-based assertions $\mathcal{A} = \{\phi_j\}$, where each assertion $\phi_j = (\text{check}_j, \text{params}_j)$ is a predicate over trajectories (e.g., \texttt{action\_exists}, \texttt{action\_before}, \texttt{final\_action}). Given the failure reservoir $\mathcal{R}$, the grader evaluates every trajectory against all assertions, producing per-assertion pass rates $\hat{r}_j = |\mathcal{R}|^{-1} \sum_{\tau \in \mathcal{R}} \mathbf{1}[\text{check}_j(\tau, \text{params}_j)]$ that quantify the prevalence of specific behavioral patterns. The grader is entirely rule-based with no LLM calls.

An LLM-based assertion analyzer $\mathcal{Z}$ then receives the pass rates, a stratified sample of trajectories from $\mathcal{R}$, and a summary of the current skill bank $\mathcal{S}$. The analyzer \textit{evolves the assertion set} $\mathcal{A}$ via operations (add, delete, modify), adapting the diagnostic criteria to the current policy's failure landscape. It also produces a \textit{failure diagnosis} $\mathcal{F}_{\text{diag}}$ with prevalence-ranked patterns and selects representative episodes $\mathcal{E}_{\text{sel}}$ for the skill optimizer.

\rparagraph{Skill Version Creation}
Rather than proposing a single skill in isolation, the creation pipeline generates a new \textit{version} of the skill library through a two-step analyst--author process. An LLM-based \textit{version analyst} $\mathcal{Z}_v$ receives the failure diagnosis $\mathcal{F}_{\text{diag}}$, the selected episodes $\mathcal{E}_{\text{sel}}$, the current skill bank $\mathcal{S}$, the history of previous version proposals and their accept/reject outcomes, and action-pattern frequency statistics from recent rollouts. It produces a \textit{change plan} $\mathcal{P}$: a set of 1--3 operations, each of type \textsc{add} (introduce a new skill), \textsc{modify} (refine an existing skill's content or trigger), or \textsc{delete} (remove an obsolete or harmful skill). The analyst prioritizes unaddressed failure patterns for additions, refines skills whose guidance contradicts successful behavior for modifications, and removes skills that the policy has internalized or that conflict with current performance for deletions. The total number of active skills is capped at $K_{\max}$ after all operations.

An LLM-based \textit{version author} $\mathcal{O}_v$ then executes the change plan $\mathcal{P}$, writing concrete skill content, trigger conditions, and grounded examples for each operation. For any \textsc{add} operation, the proposed skill must pass a \textit{trigger rate gate}: $\hat{p}_{\text{trigger}} \geq \delta$ (default $\delta = 0.1$), ensuring testability within a reasonable number of episodes. The resulting set of operations defines a candidate version $\mathcal{V}_{\text{new}}$ of the skill library.

\rparagraph{Trigger-Based Skill Injection}
At each step $t$ of a multi-turn episode, the trigger matcher determines which skills from the active version are injected into the prompt. Each skill's trigger condition $\text{trig}_k$ is a predicate over the current step index and the previous action:
\begin{equation}
    \mathcal{S}_{\text{active}}^{(t)} = \{S_k \in \mathcal{S} : \text{trig}_k(t, a_{t-1}) = 1\}.
    \label{eq:trigger}
\end{equation}
This per-step trigger mechanism allows different skills to activate at different moments within a single episode, providing targeted guidance without inflating every prompt. The specific trigger types and their semantics are detailed in Appendix~\ref{app:triggers}.


\subsection{RL-Guided Skill Evolution with Thompson Sampling}
\label{sec:skill_evolution}

As described in \S\ref{sec:design}, rollouts within each GRPO group are split between the candidate version $\mathcal{V}_{\text{new}}$ and the current version $\mathcal{V}_{\text{old}}$. The skill evolution operates in \textit{cycles} of $K_e$ training steps: at the start of each cycle, the creator pipeline (\S\ref{sec:skill_creator}) proposes a candidate version; the within-group split then evaluates it over $K_e$ steps; at the end of the cycle, the version is accepted or rejected, and the next cycle begins immediately.

The within-group allocation is a two-armed bandit problem: we must balance \textit{evaluating} which version is better against \textit{exploiting} the currently-favored version for better training outcomes. Thompson Sampling \citep{thompson1933likelihood, russo2018tutorial} achieves both naturally---it allocates more rollouts to the version that appears better (improving training efficiency) while maintaining stochastic exploration of the alternative (gathering evidence for the accept/reject decision).

\rparagraph{Version Assignment}
Each arm maintains a Beta posterior over its success probability, initialized with uniform priors $\text{Beta}(1, 1)$:
\begin{equation}
    p_{\text{new}} \sim \text{Beta}(\alpha_{\text{new}}, \beta_{\text{new}}), \quad p_{\text{old}} \sim \text{Beta}(\alpha_{\text{old}}, \beta_{\text{old}}).
    \label{eq:thompson_posteriors}
\end{equation}
At the start of each episode, the version assignment is sampled via Thompson Sampling:
\begin{equation}
    d = \mathbf{1}[p_{\text{new}} > p_{\text{old}}], \quad \text{with } \Pr(\text{new}) = \text{clamp}\!\left(\Pr(p_{\text{new}} > p_{\text{old}}),\, \varepsilon_{\text{ts}},\, 1{-}\varepsilon_{\text{ts}}\right),
    \label{eq:thompson_decision}
\end{equation}
where $\varepsilon_{\text{ts}} = 0.2$ ensures sufficient exploration of both versions~\citep{russo2018tutorial}. All steps within a single rollout use the assigned version's skill library.

\rparagraph{Adaptive Discount Posterior}
Unlike classical bandit settings where arm rewards are stationary, the policy $\pi_\theta$ evolves during each evaluation cycle: the arm success probability $p_{a,t} = P(\text{success} \mid \text{arm } a, \pi_{\theta_t})$ changes with each GRPO update. The standard Beta posterior, which pools all episodes equally, conflates data from multiple policies and introduces staleness bias. We address this with a \textit{sample-size-adaptive discount}: at each step~$t$, existing pseudo-counts are shrunk before incorporating the $m_t$ successes out of $n_t$ episodes assigned to each arm:
\begin{equation}
    \alpha \leftarrow w_t \cdot \alpha + m_t, \quad \beta \leftarrow w_t \cdot \beta + (n_t - m_t), \qquad w_t = \frac{1}{1 + n_t/M},
    \label{eq:beta_update}
\end{equation}
where $M > 0$ is a scalar \emph{memory parameter}. Written as $w_t = 1/(1 + n_t/M)$, the discount interpolates smoothly between full retention ($w_t \to 1$ when $n_t \ll M$) and aggressive forgetting ($w_t \to 0$ when $n_t \gg M$), with the crossover at $n_t = M$ ($w_t = 1/2$). The posterior remains a valid Beta distribution with this multiplicative discount, so Thompson Sampling applies directly. We estimate $M$ from data via sequential predictive-likelihood maximization on completed evaluation cycles (Appendix~\ref{app:staleness_details}), requiring no manual tuning.

\rparagraph{Accept/Reject Decision}
At the end of each cycle, the proposed version is accepted if the adaptively-discounted posterior mean of the new arm exceeds that of the old arm:
\begin{equation}
    \text{accept}(\mathcal{V}_{\text{new}}) = \mathbf{1}\!\left[\frac{\alpha_{\text{new}}}{\alpha_{\text{new}} + \beta_{\text{new}}} > \frac{\alpha_{\text{old}}}{\alpha_{\text{old}} + \beta_{\text{old}}}\right],
    \label{eq:accept}
\end{equation}
where $\alpha, \beta$ are the discounted pseudo-counts from Eq.~(\ref{eq:beta_update}). Because old evidence is downweighted, this decision reflects the skill version's value under the \textit{current} policy, not a mixture of past policies. If accepted, $\mathcal{V}_{\text{new}}$ becomes the new baseline; if rejected, the skill bank reverts to $\mathcal{V}_{\text{old}}$. In either case, the next cycle begins immediately.


\begin{algorithm}[t]
\begin{footnotesize}
\caption{\ours: RL-Guided Skill-Policy Co-Evolution}
\label{alg:scrl}
\begin{algorithmic}[1]
    \STATE \textbf{Input}: Policy $\pi_{\theta_0}$, training steps $T$, evolution frequency $K_e$.
    \STATE \textbf{Initialize}: Skill bank $\mathcal{S} {=} \emptyset$, reservoir $\mathcal{R} {\leftarrow} \emptyset$, $\text{Beta}(1,1)$ priors for $p_{\text{new}}, p_{\text{old}}$, $\hat{M} {\leftarrow} \infty$.
    \FOR{$t=1, \dots, T$}
        \STATE \textcolor{paleblue}{[\textbf{GRPO Rollout with Skill A/B Testing}]}
        \STATE Sample task $q \sim \mathcal{D}$; assign each episode to $\mathcal{V}_{\text{new}}$ or $\mathcal{V}_{\text{old}}$ via Thompson Sampling (Eq.~\ref{eq:thompson_decision}).
        \STATE Evaluate triggers; inject active skills from the assigned version into the prompt.
        \STATE Generate $\{\tau_i\}_{i=1}^G \sim \pi_{\theta_{t-1}}(\cdot | q, \mathcal{S}_{\text{active}}; \mathcal{T})$; compute rewards $\{r_i\}$.
        \STATE Update policy: $\theta_t \leftarrow \text{GRPO}(\theta_{t-1}, \{\tau_i\}, \{r_i\})$. \hfill {\color{gray}\textit{// duty 1: policy optimization}}
        \STATE Add failed episodes to reservoir $\mathcal{R}$. \hfill {\color{gray}\textit{// duty 2: failure collection}}
        \STATE Discount ($w_t {=} \hat{M}/(\hat{M}{+}n_t)$) and update Beta posteriors for $p_{\text{new}}, p_{\text{old}}$ (Eq.~\ref{eq:beta_update}). \hfill {\color{gray}\textit{// duty 3: bandit eval.}}
        \IF{$t \bmod K_e = 0$}
            \STATE \textcolor{palered}{[\textbf{Evolution Decision}]} Accept $\mathcal{V}_{\text{new}}$ if $\mathbb{E}[p_{\text{new}}] > \mathbb{E}[p_{\text{old}}]$; update $\hat{M}$ via predictive-likelihood MLE.
            \STATE \textcolor{palegreen}{[\textbf{Skill Creation}]} Analyze failures in $\mathcal{R}$; propose new $\mathcal{V}_{\text{new}}$ with add/modify/delete operations.
            \STATE Reset $\text{Beta}(1,1)$ priors; begin testing $\mathcal{V}_{\text{new}}$ vs $\mathcal{V}_{\text{old}}$.
        \ENDIF
    \ENDFOR
    \STATE \textbf{Return} optimized policy $\pi_{\theta_T}$ and skill bank $\mathcal{S}$.
\end{algorithmic}
\end{footnotesize}
\end{algorithm}

The complete procedure for \ours~is provided in Algorithm~\ref{alg:scrl}. Overall, the assertion-driven skill creator replaces ad-hoc prompt engineering with structured, quantitative failure analysis, while the version-level Thompson Sampling mechanism provides principled, data-efficient validation of skill library changes using only the data already generated by GRPO training.

\subsection{Setup}

\sparagraph{Models and benchmarks}
We evaluate \ours~on three diverse agentic benchmarks: WebShop \citep{yao2022webshop} (e-commerce navigation with partial credit scoring), multi-hop question answering with a search tool (NQ + HotpotQA), and ALFWorld \citep{shridharalfworld} (embodied household tasks). We conduct experiments using the Qwen3 model family \citep{bai2023qwen} at three scales: 4B, 8B, and 14B parameters.

\rparagraph{Baselines}
We compare \ours~against two categories of baselines. \textit{Prompt-based methods}: zero-shot \textbf{Baseline} (Qwen3-4B-Instruct), and retrieval-augmented variants including \textbf{BM25}, \textbf{Contriever}, \textbf{Qwen3 Embedding}, \textbf{HippoRAG}, \textbf{MemoRAG}, \textbf{Self-RAG}, \textbf{GraphRAG}, and \textbf{Long Context} (full trajectory history in context). \textit{RL methods}: \textbf{GRPO}, standard GRPO training with a static instruction and no skills. We also evaluate ablation variants of \ours~(see \S\ref{sec:ablation}).

\rparagraph{Implementation}
We build on the VeRL framework \citep{sheng2025hybridflow} with Ray-based distributed training on 8$\times$A100 GPUs. For the skill creator pipeline, we use Claude Sonnet 4.5 as the LLM for both the assertion analyzer and skill optimizer. Thompson Sampling uses $\text{Beta}(1,1)$ priors, evolution frequency $K_e = 5$ steps, and exploration bound $\varepsilon_{\text{ts}} = 0.2$. The failure reservoir maintains up to 200 trajectories with 70/30 failure/success stratification.

\section{Related Work}
\label{sec:related}
%=============================================================================

\sparagraph{Reinforcement Learning for LLMs}
Reinforcement learning has become a powerful paradigm for post-training LLMs, from alignment with human preferences \citep{ouyang2022training, rafailov2023direct} to reasoning via verifiable rewards. DeepSeek-R1 \citep{guo2025deepseek} demonstrated the effectiveness of RLVR, where GRPO \citep{shao2024deepseekmath} enables group-wise advantage estimation without a critic model. DAPO \citep{yu2025dapo} introduces training stability improvements such as clip-higher and dynamic sampling. Dr.~GRPO \citep{liu2025understanding} rectifies length bias. \ours~is orthogonal to these algorithmic improvements and can be integrated with any GRPO-based training pipeline.

\rparagraph{Tool-Integrated Agents}
LLM agents interact with environments through tool use \citep{yao2023react}. IRCoT \citep{trivedi-etal-2023-interleaving} interleaves chain-of-thought reasoning \citep{wei2022chain} with information retrieval. Toolformer \citep{schick2023toolformer} teaches tool usage via SFT. The RL paradigm has enabled multi-turn tool interactions \citep{feng2025retool, li2025torl}, with search tools excelling at question answering \citep{jin2025search, song2025r1}. End-to-end RL frameworks for tool use \citep{xue2025simpletir, jiang2025verltool} have grown rapidly, yet the role of \textit{dynamic skill guidance} during RL training has been largely unexplored: all existing approaches rely on static instructions designed before training begins.

\rparagraph{Skill and Instruction Optimization}
The performance of LLM agents is highly sensitive to their guiding instructions \citep{zhou2025multi}. Automated prompt optimization (APO) approaches use paraphrasing \citep{zhou2023large}, LLM-based optimizers \citep{yang2024large}, or textual gradients \citep{pryzant-etal-2023-automatic, 2025textgrad} to improve instructions, but operate offline. INSPO \citep{zhou2025inspo} is the first to integrate instruction optimization into the RL loop via a population-based approach with softmax-weighted sampling and successive halving \citep{jamieson2016non}. In the skill domain, Reflexion \citep{shinn2023reflexion} accumulates verbal feedback, ExpeL \citep{zhao2024expel} extracts experiential rules, MemRL \citep{zhang2026memrl} evolves episodic memory, and SkillRL \citep{xia2025skillrl} distills teacher-model skills for RL augmentation. Anthropic's Skill Creator \citep{anthropic2025skillcreator} pioneered the automated creation of modular, trigger-based skills from agent experience. \ours~builds on this paradigm but replaces the human judge with RL reward signals and integrates skill creation directly into the training loop, enabling online co-evolution rather than static augmentation.

\rparagraph{Bandit Methods for Online Selection}
Thompson Sampling \citep{thompson1933likelihood} is a Bayesian approach to the multi-armed bandit problem that maintains posterior distributions over arm rewards and samples actions proportionally to their probability of being optimal \citep{russo2018tutorial}. In the context of RL-based agent training, INSPO \citep{zhou2025inspo} uses softmax-weighted sampling over an instruction population. \ours~instead employs per-skill Thompson Sampling with trigger-gated counting: each skill maintains independent Beta posteriors updated only from episodes where its trigger fires, providing a natural handle for the heterogeneous activation patterns of different skill types.

\label{sec:conclusion}

We introduced \ours, a framework that enables the co-evolution of modular skills and policy learning for agentic RL. By integrating an assertion-driven skill creator pipeline with Thompson Sampling-based skill evolution directly into the GRPO training loop, \ours~achieves substantial convergence speedup with zero additional rollouts. The triple-duty design, where each GRPO rollout simultaneously provides policy gradients, failure data for skill creation, and bandit evaluation signals for skill evolution, demonstrates that significant efficiency gains are possible by extracting more value from existing training computation.

\section{Adaptive Staleness Correction: Details}
\label{app:staleness_details}

This appendix motivates the adaptive discount rule $w_t = 1/(1 + n_t/M)$ from Eq.~(\ref{eq:beta_update}), grounds it in the Bayesian posterior update, and provides the full procedure for estimating~$M$ from data.

\subsection{Motivation: From Bayesian Updates to Adaptive Discounting}

We begin from the standard Beta-Bernoulli posterior update and show how the discount rule emerges naturally from the desire to control the influence of new observations relative to accumulated evidence.

\rparagraph{The Posterior Mean as a Weighted Average}
After observing $m_t$ successes in $n_t$ episodes at step~$t$, the standard (undiscounted) Beta posterior update $\alpha_t = \alpha_{t-1} + m_t$ yields a posterior mean that is a weighted average of the prior estimate and the new observation:
\begin{equation}
    \mu_t = \frac{S_{t-1}}{S_{t-1} + n_t}\;\mu_{t-1} + \frac{n_t}{S_{t-1} + n_t}\;\frac{m_t}{n_t},
    \label{eq:standard_update}
\end{equation}
where $S_{t-1} = \alpha_{t-1} + \beta_{t-1}$ is the total pseudo-count. The weight on new evidence is $n_t/(S_{t-1} + n_t)$: the more data observed so far ($S_{t-1}$ large), the less each new batch shifts the estimate. In a stationary setting, this is optimal---the posterior concentrates as evidence accumulates.

\rparagraph{The Problem Under Non-Stationarity}
When the policy evolves, the success probability $p_{a,t}$ drifts across steps. The accumulated pseudo-count $S_{t-1}$ reflects all past policies, not just the current one. A large $S_{t-1}$ (from many past observations) anchors the estimate to stale data, making it slow to respond to genuine changes in the success rate. The fundamental issue is that the posterior treats all evidence as equally valid, regardless of when it was collected.

\rparagraph{Discounting as Effective Sample Size Control}
A geometric discount $\alpha_t = w \cdot \alpha_{t-1} + m_t$ shrinks the accumulated pseudo-count before each update, reducing the effective sample size of old evidence. With discount factor~$w$, the posterior mean becomes:
\begin{equation}
    \mu_t = \frac{w \cdot S_{t-1}}{w \cdot S_{t-1} + n_t}\;\mu_{t-1} + \frac{n_t}{w \cdot S_{t-1} + n_t}\;\frac{m_t}{n_t}.
    \label{eq:discounted_update}
\end{equation}
The key quantity is the ratio of old-to-new influence: $w \cdot S_{t-1} : n_t$. With a constant~$w$, this ratio depends on the entire allocation history through~$S_{t-1}$. In our setting, Thompson Sampling causes the per-arm episode count $n_t$ to vary by 5--10$\times$ across steps (e.g., 168 to 1112 in Search). The same constant~$w$ cannot be appropriate for both: after a step where the arm received 168 episodes, the accumulated estimate is noisy and should be retained; after a step with 1112 episodes, the new observation is precise and old evidence should be discounted more aggressively.

\rparagraph{The $M$-Parameterization}
To decouple the discount from the allocation history, we set $w_t = M/(M + n_t) = 1/(1 + n_t/M)$. Substituting into Eq.~(\ref{eq:discounted_update}), the effective old pseudo-count after discounting becomes $w_t \cdot S_{t-1} = M \cdot S_{t-1}/(M + n_t)$. This yields an intuitive interpretation:
\begin{itemize}[itemsep=2pt, leftmargin=*]
    \item When $n_t = M$ (the crossover): $w_t = 1/2$, and old and new evidence contribute equally.
    \item When $n_t \ll M$ (minority arm, noisy observation): $w_t \approx 1$, the accumulated estimate is mostly retained.
    \item When $n_t \gg M$ (majority arm, precise observation): $w_t \approx 0$, old evidence is aggressively discounted.
\end{itemize}
The parameter $M$ can be interpreted as the \emph{number of new observations at which old evidence loses half its weight}. It controls the balance between responsiveness to drift and robustness to noise, from a single scalar.

\rparagraph{Connection to Exponential Smoothing}
The update $\alpha_t = w_t \cdot \alpha_{t-1} + m_t$ with $w_t = M/(M + n_t)$ can be rewritten as
\begin{equation}
    \alpha_t = \frac{M}{M + n_t} \cdot \alpha_{t-1} + m_t = \frac{M \cdot \alpha_{t-1} + m_t \cdot (M + n_t) / 1}{M + n_t}.
\end{equation}
When $n_t = n$ is constant, this reduces to a standard geometric discount $w = M/(M+n)$, recovering exponential smoothing with a fixed forgetting rate. The generalization to variable $n_t$ preserves the exponential smoothing structure while adapting the forgetting rate to the informativeness of each observation.

\rparagraph{Why Not Estimate $w$ Directly?}
An alternative is to estimate a constant discount factor~$w^*$ by minimizing a worst-case MSE expression derived from per-step KL divergence bounds (as in importance-weighted approaches). This requires estimating the drift rate, assumes constant~$n$, and introduces a scaling parameter to handle the looseness of Pinsker's inequality. The $M$-parameterization avoids these issues: it handles variable~$n_t$ naturally and is estimated purely from observational data via the predictive likelihood described below.

\subsection{Predictive-Likelihood Criterion}

Suppose we have $K$ completed A/B tests, each spanning $T$ training steps. Test~$k$ provides per-step per-arm data $\{(m_t^{(k,a)}, n_t^{(k,a)})\}_{t=1}^T$ for each arm $a \in \{A, B\}$, where $m_t$ is the number of successes and $n_t$ is the number of episodes assigned to arm~$a$ at step~$t$.

For a candidate $M$ and one arm's data sequence within one test, the \textbf{sequential predictive log-likelihood} is:
\begin{enumerate}[itemsep=2pt]
\item Initialize $\alpha = 1$, $\beta_p = 1$ \quad ($\mathrm{Beta}(1,1)$ prior).
\item For $t = 1, \ldots, T$:
  \begin{enumerate}
    \item Discount: $\tilde\alpha = \frac{M}{M + n_t} \cdot \alpha$, \; $\tilde\beta = \frac{M}{M + n_t} \cdot \beta_p$.
    \item Score: $\ell_t = \log P\bigl(m_t \mid \mathrm{BetaBin}(n_t, \tilde\alpha, \tilde\beta)\bigr)$.
    \item Update: $\alpha = \tilde\alpha + m_t$, \; $\beta_p = \tilde\beta + (n_t - m_t)$.
  \end{enumerate}
\item Return $\sum_{t=1}^T \ell_t$.
\end{enumerate}
The BetaBinomial predictive probability at step~2(b) is $P(m \mid n, \tilde\alpha, \tilde\beta) = \binom{n}{m} B(m + \tilde\alpha, n - m + \tilde\beta) / B(\tilde\alpha, \tilde\beta)$, where $B(\cdot, \cdot)$ is the Beta function. The total log-likelihood is
\begin{equation}
    L(M) = \sum_{k=1}^K \sum_{a \in \{A, B\}} \mathrm{score}\bigl(M;\, \{m_t^{(k,a)}, n_t^{(k,a)}\}_{t=1}^T\bigr),
    \label{eq:total_ll}
\end{equation}
and the MLE is $\hat{M} = \arg\max_M L(M)$, found by grid search over $M \in [1, 2000]$.

\subsection{Online Estimation}

The log-likelihood~(\ref{eq:total_ll}) decomposes additively across tests. After each completed test~$k$, we update $L_k(M) = L_{k-1}(M) + \mathrm{score}(M;\, \text{test}~k, A) + \mathrm{score}(M;\, \text{test}~k, B)$ and re-compute $\hat{M}_k = \arg\max_M L_k(M)$. This is $O(|\text{grid}| \times T)$ work per new test, with no need to re-score history.

\subsection{Deployment Matching}

The scoring procedure mirrors the actual Thompson Sampling deployment:
\begin{itemize}[itemsep=2pt, leftmargin=*]
    \item The prior resets at the start of each test (as in deployment).
    \item The discount $w_t = M/(M + n_t)$ uses the \emph{actual} per-arm allocation at each step, not an average or fixed~$n$.
    \item The BetaBinomial score evaluates one-step-ahead prediction accuracy---the same quantity that determines posterior quality during the test.
\end{itemize}
This avoids the model misspecification that arises from estimating~$M$ using the overall success rate with a fixed per-arm count. In our experiments, such a ``naive'' approach overestimates $M$ by 1.5--2.2$\times$ because the overall rate smooths out per-arm volatility, causing the MLE to infer slower drift than actually exists.
